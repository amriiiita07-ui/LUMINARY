## ‚ú¶ What is LUMINARY?

**LUMINARY** is a production-grade, full-stack **Urban Intelligence & City Pulse Analytics Platform** designed to ingest heterogeneous smart-city data streams and transform them into decision-grade, predictive intelligence dashboards.

Built as a showcase project for data science and full-stack engineering portfolios, LUMINARY demonstrates **advanced SQL architecture**, **real-world analytics patterns**, and **cinematic UI/UX design** ‚ÄĒ the kind of work that makes senior engineers and recruiters stop scrolling.

### Key SQL Patterns Used

```sql
-- 1. Window Functions
AVG(mobility_score) OVER (PARTITION BY zone_id ORDER BY day ROWS BETWEEN 6 PRECEDING AND CURRENT ROW)

-- 2. Recursive CTEs
WITH RECURSIVE hierarchy AS ( ... UNION ALL ... )

-- 3. Cohort Retention
ROUND(retained_users * 100.0 / FIRST_VALUE(users) OVER (PARTITION BY cohort_month ORDER BY month_number), 1)

-- 4. Percentile Ranking
DENSE_RANK() OVER (ORDER BY risk_score DESC)
PERCENT_RANK() OVER (ORDER BY anomaly_count DESC)

-- 5. Statistical Z-Score (in Trigger)
v_zscore := (NEW.mobility_score - v_mean) / v_stddev;

-- 6. Composite Risk Scoring
ROUND((volatility/avg_mobility * 35 + anomaly_density * 45 + critical_pct * 20)::numeric, 2)
```

## ‚ú¶ Getting Started

### Prerequisites

Make sure you have these installed:

```bash
node --version   # v18.0.0 or higher
npm --version    # v9.0.0 or higher
git --version    # Any recent version
```

You'll also need a **PostgreSQL database**. Free options:
- [Neon](https://neon.tech) ‚Üź Recommended (free tier, serverless)
- [Supabase](https://supabase.com) (free tier)
- [Railway](https://railway.app) (free trial)
- Local PostgreSQL installation

---

### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/luminary.git
cd luminary
```

---

### 2. Install Dependencies

```bash
npm install
```

---

### 3. Configure Environment Variables

```bash
cp .env.example .env.local
```

Open `.env.local` and fill in your values:

```env
# Get this from Neon / Supabase / Railway dashboard
DATABASE_URL="postgresql://username:password@hostname:5432/luminary?sslmode=require"

# Generate with: openssl rand -base64 32
NEXTAUTH_SECRET="your-generated-secret-here"
NEXTAUTH_URL="http://localhost:3000"

NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

---

### 4. Setup the Database

```bash
# Generate Prisma client from schema
npm run db:generate

# Push schema to your database (creates all tables)
npm run db:push

# Seed with realistic demo data (100k+ records)
npm run db:seed
```

> ‚Ź≥ The seed script inserts ~100,000 records across mobility events, air quality logs, anomalies, risk snapshots and sentiment logs. This takes 30‚Äď60 seconds.

---

### 5. Apply SQL Analytics Layer

Connect to your database and run the analytics SQL:

```bash
# If using psql locally:
psql $DATABASE_URL < sql/analytics/analytics.sql

# If using Neon / Supabase:
# Paste the contents of sql/analytics/analytics.sql into the SQL editor
```

This creates:
- 8 analytics views (`v_zone_risk_intelligence`, `v_daily_mobility_summary`, etc.)
- 1 stored procedure (`refresh_risk_snapshots()`)
- 1 anomaly detection trigger (`trg_mobility_anomaly`)
- 6 performance indexes

---

### 6. Start the Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) ūüéČ

| Route | Description |
|---|---|
| `/` | Landing page ‚ÄĒ hero, features, SQL showcase |
| `/dashboard` | Main analytics console ‚ÄĒ KPIs, zones, anomalies |

---

### 7. Explore the Database (Optional)

```bash
npm run db:studio
```

Opens **Prisma Studio** at [http://localhost:5555](http://localhost:5555) ‚ÄĒ a beautiful visual database explorer.

## ‚ú¶ Deployment

### Deploy to Vercel (Recommended)

**Step 1** ‚ÄĒ Create a Neon database at [neon.tech](https://neon.tech)

**Step 2** ‚ÄĒ Import your GitHub repo to [vercel.com/new](https://vercel.com/new)

**Step 3** ‚ÄĒ Add environment variables in Vercel dashboard:

```
DATABASE_URL        ‚Üí  your Neon connection string
NEXTAUTH_SECRET     ‚Üí  openssl rand -base64 32
NEXTAUTH_URL        ‚Üí  https://your-project.vercel.app
NEXT_PUBLIC_APP_URL ‚Üí  https://your-project.vercel.app
```

**Step 4** ‚ÄĒ Deploy! Vercel auto-detects Next.js.

**Step 5** ‚ÄĒ Run migrations on your production database:

```bash
# In Vercel dashboard ‚Üí Settings ‚Üí Environment ‚Üí copy DATABASE_URL
# Then locally:
DATABASE_URL="your-production-url" npx prisma db push
DATABASE_URL="your-production-url" npm run db:seed
```

**Step 6** ‚ÄĒ Apply SQL analytics layer via Neon SQL editor:
```
Paste contents of sql/analytics/analytics.sql ‚Üí Run
```

## ‚ú¶ API Reference

All endpoints are under `/api/` and return JSON.

### `GET /api/analytics`
Returns city-wide KPI summary.

```json
{
  "mobilityIndex": 87.4,
  "mobilityDelta": 3.2,
  "openAnomalies": 6,
  "criticalAlerts": 2,
  "zonesMonitored": 11,
  "computedAt": "2025-01-15T10:30:00.000Z"
}
```

### `GET /api/zones`
Returns zone risk rankings.

```json
{
  "zones": [
    {
      "zoneId": "clxxx...",
      "zoneName": "Downtown Core",
      "district": "Central Borough",
      "riskScore": 82.4,
      "riskTier": "CRITICAL",
      "weeklyDrift": 7.2,
      "anomalyCount": 4,
      "avgMobility": 88.1
    }
  ],
  "total": 11
}
```

### `GET /api/mobility?zoneId=<id>`
Returns 7-day mobility trend. `zoneId` is optional (defaults to city-wide).

```json
{
  "trend": [
    { "day": "2025-01-09", "avgScore": 68.2, "pedestrians": 42000, "vehicles": 18000 }
  ]
}
```

### `GET /api/anomalies?limit=<n>`
Returns anomaly feed, newest first.

```json
{
  "anomalies": [
    {
      "id": "clxxx...",
      "signalType": "MOBILITY",
      "zScore": 3.82,
      "severity": "HIGH",
      "resolved": false,
      "detectedAt": "2025-01-15T08:22:00.000Z",
      "zone": { "name": "Harbor Edge", "district": { "name": "Harbor Quarter" } }
    }
  ],
  "total": 8
}
```
